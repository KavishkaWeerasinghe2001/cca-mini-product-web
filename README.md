## Product Landing Page

A responsive product landing page built with HTML, CSS, and JavaScript.
This lab focuses on creating a clean UI, managing state in JavaScript, rendering content from data, and handling user interactions.

## Project Overview

This project is a mini product website for **CCA**. It includes a landing page with a header, hero section, feature tabs, pricing cards, and a signup form.

The goal of this lab is to practice front-end fundamentals while thinking in a component-based and state-driven way, similar to how modern frameworks like React or Next.js work.

## Features

* Responsive navigation menu
* Hero section with call-to-action buttons
* Live active user counter
* Feature tabs with dynamic content rendering
* Monthly and annual pricing toggle
* Pricing cards rendered from JavaScript data
* Signup form with inline validation messages
* LocalStorage support for remembering user preferences
* Smooth scrolling to page sections
* Responsive layout for mobile and desktop screens

## Tech Stack

* HTML5
* CSS3
* JavaScript
* LocalStorage API


## Main Sections

### Header

The header contains the CCA logo, navigation links, and a mobile menu button.

### Hero

The hero section introduces the product with two call-to-action buttons:

* Get Started
* See Features

It also includes a live user counter that updates automatically.

### Features

The features section uses tab-based switching. Users can select between:

* Speed
* Quality
* Scale

The content is rendered dynamically using JavaScript.

### Pricing

The pricing section contains three plans:

* Starter
* Pro
* Team

Users can switch between monthly and annual billing. Pricing cards are generated from JavaScript data instead of being hardcoded directly in HTML.

### Signup Form

The signup form collects:

* Name
* Email
* Track

It includes inline validation and displays a message after interaction.

## Key JavaScript Concepts Practiced

* State management using a `state` object
* Rendering UI from data
* Event listeners
* DOM manipulation
* Form validation
* LocalStorage
* Conditional rendering
* Reusable helper functions
